# WCAG by User Group

---

*Ava had been staring at the WCAG quick reference for ten minutes. "I can see how it's organized — principles, guidelines, criteria. But when I'm reviewing a design, I don't think in principles. I think about who's going to use it."*

*"Same," Jordan said. "When I'm testing, I want to know: what do I check for screen reader users? What do I check for keyboard users? I don't want to scan all four principles every time."*

*Priya nodded. "What if we flipped the map? Instead of starting with the spec and working toward people, start with the people and work toward the criteria."*

*"Like a cheat sheet?" Marcus asked.*

*"Like a lens," Priya said. "Same criteria, different way in."*

---

## A Different Way to Read WCAG

The [Intopia WCAG 2.2 Map](https://intopia.digital/wp-content/uploads/2023/12/Intopia-WCAG-2.2-Map-Landscape-version.pdf) is an excellent visual reference that organizes every success criterion by principle and guideline — the way WCAG itself is structured. It answers the question: **"What does WCAG say?"**

This page answers a different question: **"Who does each criterion help?"**

The same criteria appear here, but organized by the six user groups from [People & Assistive Devices](/tutorial/people-and-devices). Many criteria show up under multiple groups — that's not a mistake. It reflects how interconnected accessibility is. Fixing contrast helps low vision users *and* people with color vision differences. Good heading structure helps screen reader users *and* people with cognitive differences.

Conformance levels are marked: **A** (minimum), **AA** (standard target), **AAA** (aspirational). Criteria new in WCAG 2.2 are noted.

## Using Both Maps Together

The [Intopia WCAG 2.2 Map](https://intopia.digital/wp-content/uploads/2023/12/Intopia-WCAG-2.2-Map-Landscape-version.pdf) and this people-based map are two views of the same information:

- **Intopia map** — start with WCAG, find the criteria you need to meet. Best for compliance reviews and audits.
- **People-based map** — start with who you're building for, find the criteria that matter most. Best for design reviews, testing plans, and building team empathy.

Neither replaces the other. Use the Intopia map when you need to verify conformance. Use this map when you need to understand *why* a criterion matters and *who* it serves.
